I built three Python exercises demonstrating control flow and loops: a personal info calculator with retirement and coffee budget calculations, a grade classification system using conditional logic, and an interactive multiplication table generator with input validation.

I got tripped up initially on proper input validation in exercise3 and formatting the multiplication table with aligned columns, but resolved these by using range validation and string formatting techniques.

If I were to do this differently, I would add more comprehensive error handling and make the code more modular by separating the UI logic from the core calculation functions.
